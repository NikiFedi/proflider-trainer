# Тренажёр «Студенческий лидер»

Веб-тренажёр для подготовки к конкурсу: банк вопросов по нормативным актам, разложенный по группам, с автопроверкой (варианты, да/нет, открытые вопросы с самооценкой).

## Структура проекта

```
proflider/
├── index.html
├── package.json
├── vite.config.js
└── src/
    ├── main.jsx              — точка входа
    ├── App.jsx               — вся логика интерфейса (не трогать без необходимости)
    └── data/
        ├── group1_profsoyuz.js      — Устав, СКС, ФЗ-10
        ├── group2_obrazovanie.js    — ФЗ-273, ГИА, академотпуск, переводы, дипломы
        ├── group3_obshezhitiya.js   — общежития (коммуналка, санитария, письма)
        ├── group4_stipendii.js      — стипендии (приказ №1663)
        ├── group5_disciplina.js     — дисциплинарные взыскания (№185)
        ├── rosmolodezh.js           — Росмолодёжь.Гранты, номинации, логотипы
        ├── daty.js                  — памятные даты
        ├── profkom_vshte.js         — тест 50 вопросов (Профком ВШТЭ 2026)
        ├── proshlye_etapy.js        — сборные тесты прошлых этапов конкурса
        └── groups.js                — собирает всё выше в единый список GROUPS
```

## Запуск локально

```
npm install
npm run dev
```

Откроется адрес вида `http://localhost:5173`.

## Как добавить новый тест (когда присылаешь документ)

1. Определяем, к какой группе относится документ (Профсоюз/СКС, Образование, Общежития, Стипендии, Дисциплинарные, Росмолодёжь, Даты, Профком, Прошлые этапы) — либо создаём новую группу.
2. В соответствующий файл в `src/data/` добавляется новый объект теста:

```js
{
  id: "unique_id",              // латиницей, без пробелов
  title: "Название теста",
  subtitle: "Краткое описание — N вопросов",
  questions: [
    { type: "mc", q: "Текст вопроса?", options: ["A", "B", "C", "D"], correct: 0 },
    { type: "tf", q: "Утверждение для оценки да/нет.", correct: true, note: "Пояснение/ссылка на статью." },
    { type: "open", q: "Открытый вопрос?", answer: "Ответ, который надо запомнить." },
  ],
}
```

Три типа вопросов:
- `mc` — выбор одного варианта из 4 (`options` + индекс `correct`, считая с 0);
- `tf` — да/нет (`correct: true/false`, `note` — короткое пояснение с ссылкой на статью/пункт);
- `open` — открытый вопрос с ответом, который показывается по кнопке, а пользователь сам отмечает "знал/не знал".

3. Если нужна новая группа целиком — создать новый файл `src/data/имя.js` с `export const MY_TESTS = [...]`, импортировать его в `groups.js` и добавить новую запись в массив `GROUPS`.

## Публикация на GitHub

1. Создать пустой репозиторий на github.com (без README/лицензии, чтобы не было конфликтов).
2. Локально (или через GitHub Desktop):

```
cd proflider
git init
git add .
git commit -m "Тренажёр Студенческий лидер: первая версия"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
```

3. Для просмотра тренажёра как сайта — можно включить **GitHub Pages** (Settings → Pages → source: GitHub Actions, шаблон "Static Site" под Vite), либо задеплоить на Vercel/Netlify, подключив репозиторий (build command `npm run build`, output `dist`).

## Как мы будем работать дальше

Присылаешь документ (приказ, ФЗ, устав, методичку и т.п.) → говоришь, в какую группу это идёт (или новую) → получаешь готовый файл с тестами в этом же формате, который останется только добавить в репозиторий (импорт в `groups.js` + `git add/commit/push`, либо просто заменить/добавить файл через веб-интерфейс GitHub).
